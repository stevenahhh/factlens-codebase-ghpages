결과 정리

✓

주장 추출 동작

업로드한 문서에서 검증 가능한 주장 문장만 분리해 JSON으로 정규화됨. 무관한 서술은 제외됨.

✓

3단계 근거 수집 동작

Bedrock KB 검색 → 인터넷 보완 → 내장 자료 순서로 폴백이 실제로 작동함. 근거 없이 판정하는 경우 없음.

✓

4가지 라벨 판정 출력

근거 있음 / 충돌 / 부족 / 과장 표현으로 결과가 일관되게 나옴. 판정 라벨이 고정되어 후처리 가능.

✓

검증 리포트 저장 및 조회

분석 결과가 DynamoDB에 저장되고, 고유 ID로 다시 불러올 수 있음.

✓

프론트엔드 연동

CloudFront → API Gateway → Lambda 경로로 업로드부터 결과 조회까지 UI에서 동작함.

△

Bedrock 파라미터 바인딩

temperature, top_p 등 추론 파라미터는 런타임 연동 미완료 상태. 모델 연결 완료 시 적용 예정.
